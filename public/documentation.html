Return all movies: get '/movies'

This endpoint uses GET to return a json with all the movies in this API and some data about them.
There are no parameters in the url, and no data expected in the request body.

Return movie data: get '/movies/:title'

This endpoint uses GET to return a json with data about a specific movie.
It requires the name of the requested movie in the url, but there is no expected data in the request body.

Return genre data: get '/genres/:genre'

This endpoint uses GET to return text detailing specifics about the requested genre.
It requires a genre type in the url, but there is no expected data in the request body.

Return director data: get '/directors/:name'

This endpoint uses GET to return a json with data about a specific director.
The URL requires the director's name, but there is no expected data in the request body.

Allow user registration: post '/users/register/:username' request json

This endpoint uses POST to add a user. It will verify that this has happened by returning the username and an id.
The URL requires a username. This username must also be present in a json object.

{ "name": "HannahHogan", "password": "Pa55w0rd" }

Allow user info update: put '/users/:username/:newUsername'

This endpoint uses PUT to allow users to update their usernames. It will verify that this has happened by returning confirmation text.
The URL Requires both the old username and the new username, but there is no expected data in the request body.

Allow users to add favorite movie: put '/users/:username/favorites/:title/add

This endpoint uses PUT to add a movie to a user's favorites list. It will verify that this has happened by returning confirmation text.
The URL requires both the user's username and the title of the movie, but there is no expected data from the request body.

Allow users to remove favorite movie: delete '/users/:username/favorites/:title/remove'

This endpoint uses DELETE to remove movies from their favorites list. It will verify that this has happened by returning confirmation text.
The URL requires both the user's username and the title of the movie, but there is no expected data from the request body.

Allow users to deregester: delete '/users/:username/delete'

This endpoint uses DELETE to deregister a user. It will verify that this has happened by returning confirmation text.
The URL requires the user's username, but there is no expected data from the request body.
