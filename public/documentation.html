There are no parameters in the url, and no data expected in the request body. The response will only include a death year for the director if the director is dead!

Allow user registration: post '/users' request json

This endpoint uses POST to add a user. It will verify that this has happened by returning the User json with an id.
A json object is expected in the following format: Expected JSON in this format:

{ "Username": "Hannah", "Password": "String", "Email": "hannah@website.com", "Birthday": "1991-01-01" }

Expected Response: { "Username": "Hannah", "Password": "String", "Email": "hannah@website.com", "Birthday": "1991-01-01T00:00:00.000Z", "FavMovies": [], "_id": "66c78a634935a7632b286ae3", "__v": 0 }

Allow user info update: put '/users/:username'

This endpoint uses PUT to allow users to update their user information. It will verify that this has happened by returning the updated json.
The URL Requires the current username, and the request body expects a json in the following format (with the new information):

{ "Username": "Hannah", "Password": "NewPassword", "Email": "hannah@website.org", "Birthday": "1992-02-02"}

Expected Response: { "_id": "66c63df7fc235a7f1462c1f5", "Username": "Hannah", "Password": "NewPassword", "Email": "hannah@website.org", "Birthday": "1992-02-02T00:00:00.000Z", "FavMovies": [], "__v": 0 }

Allow users to add favorite movie: POST '/users/:username/movies/:movieID

This endpoint uses POST to add a movie to a user's favorites list. It will verify that this has happened by returning the user json with the added movie in the FavMovies array.
The URL requires both the user's username and the ID of the movie, but there is no expected data from the request body.

Expected Response: { "_id": "66c63df7fc235a7f1462c1f5", "Username": "Hannah", "Password": "NewPassword", "Email": "hannah@website.org", "Birthday": "1992-02-02T00:00:00.000Z", "FavMovies": [ "66b90f36cd0e090e39228fb5" ], "__v": 0 }

Allow users to remove favorite movie: delete '/users/:username/movie/:movieID/'

This endpoint uses DELETE to remove movies from a user's favorites list. It will verify that this has happened by returning the user json with the movie removed from the FavMovies array.
The URL requires both the user's username and the title of the movie, but there is no expected data from the request body.

Expected Response: { "_id": "66c63df7fc235a7f1462c1f5", "Username": "Hannah", "Password": "NewPassword", "Email": "hannah@website.org", "Birthday": "1992-02-02T00:00:00.000Z", "FavMovies": [], "__v": 0 }

Allow users to deregester: delete '/users/:username'

This endpoint uses DELETE to deregister a user. It will verify that this has happened by returning confirmation text.
The URL requires the user's username, but there is no expected data from the request body.

Expected Response: Either "User was deleted" or "User was not found"
